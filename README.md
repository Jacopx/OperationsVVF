# OperationsVVF

A Python tool for parsing and importing Italian *Vigili del Fuoco* (fire brigade) intervention reports into a MariaDB database.

The source data is an XML report generated by Oracle Reports, exported annually. Each file contains all interventions for a given year, including vehicle dispatches and timing for each operation.

---

## Requirements

- Python 3.10+
- MariaDB / MySQL
- `mysql-connector-python`

```bash
pip install mysql-connector-python
```

---

## Configuration

Create a `config.ini` file in the project root:

```ini
[DEFAULT]
USER = your_db_user
PWD  = your_db_password
HOST = localhost
PORT = 3306
DB   = your_db_name
```

## File structure

```
.
├── main.py
├── operation.py
├── starts.py
├── config.ini
└── XML/
    └── <db_name>/
        ├── 2019.xml
        ├── 2020.xml
        └── ...
```

Place the yearly XML reports under `XML/<db_name>/` named by year (e.g. `2024.xml`).

## Database schema

```sql
CREATE TABLE Operations (
    ID       INT,
    year     VARCHAR(4),
    opn      VARCHAR(255),
    date     DATE,
    dt_exit  DATETIME,
    dt_close DATETIME,
    typology VARCHAR(500),
    x        VARCHAR(255),
    y        VARCHAR(255),
    loc      VARCHAR(255),
    boss     VARCHAR(255),
    PRIMARY KEY (ID, year)
);

CREATE TABLE Starts (
    OpID       INT,
    ID         INT,
    year       VARCHAR(4),
    vehicle    VARCHAR(255),
    exit_dt    DATETIME,
    inplace_dt DATETIME,
    back_dt    DATETIME,
    boss       VARCHAR(255),
    PRIMARY KEY (OpID, ID, year)
);
```

## Usage

```bash
python main.py <year> <0|1>
```

| Argument | Description |
|---|---|
| `year` | Four-digit year to import (e.g. `2024`) |
| `0` | Parse only — prints operations to stdout, no DB write |
| `1` | Parse and write to MariaDB |

**Examples:**

```bash
# Dry run — parse and print only
python main.py 2024 0

# Full import into the database
python main.py 2024 1
```

When writing to the DB, existing records for that year are deleted before the new data is inserted, so re-running is safe. All rows are inserted in a single batched transaction — if anything fails the whole import is rolled back.

## XML format compatibility

The tool handles two XML formats produced by different versions of the Oracle Reports export:

| Years | Operation tag | Starts tag |
|---|---|---|
| up to 2019 | `G_1` | `G_INTERVENTO1` |
| 2020 onwards | `G_RICHIEDENTE` | `G_FLAG_ANNULLA` |

Field names inside each record are consistent across all years.

---

## Typology normalisation

Raw typology strings from the XML (e.g. `INCENDIO CANNA FUMARIA`) are normalised into a fixed set of categories via keyword matching. The full mapping is defined in `TYPOLOGY_RULES` in `main.py` and covers categories including:

`Incendio`, `Bonifica`, `Alberi Pericolanti`, `Recupero Animali`, `Incidente Stradale`, `Dissesto Statico`, `Soccorso Persone`, `Fuga Gas`, `Danni Acqua`, `Apertura Alloggio`, `Rimozione Pericolanti`, and more. Unmatched typologies fall back to `Altro`.
